Permissions

For me, it is crucial to share only some of the folders - give permissions to group all
But some closest friends should have access to everything - give permissions only to specific accounts or groups
Also, I want to share some folders to strangers on social network or through chat - share via passwords (more info in passwords folder)

User permissions

Do you want share some folder only with your significant other? Easy, just look at the examples in .pmg_perms in the root folder.
Note that the file is hidden, and therefore it will not show in the gallery.
Sorry, there is no way how to show a live demo of this :( You have to install and test this application by yourself.

Groups

Except generic groups "all", "logged" and "non-logged" which can't be deleted and users are assigned automatically you can create your own custom groups.
For example create groups like family, friend or coworkers and assign users into these groups.

Security

All folders and files are visible and accessible ONLY if you have permission for them. This means that even if you know the URL, you cannot access them, no matter if it is a folder, image, audio, header or any other file.
These are direct links, that will work only to:

  • logged users
  • non-logged users
  • all users (so this link will always work)

You can easily test this out by logging in and out. Also in this folder there are three folders in total but you will be able to see only two of them at once because of the same rules.

Technical details

Permissions are saved in database via Knex which support multiple DB, including SQLite 3 which is default.

Currently there is no other way to add new user, group or passwords permission than manually edit tables in your DB :(
You can support issue on GitHub by giving thumbs-up which will increase it's priority.
